Debug

To debug, you may:

  1. Check console log see whether got error message
  2. Check superadmin see api is it call successfully (and make sure data pass correctly)
  3. Check reqDefault have the command or not
  4. Check data you pass is it same with trello
  5. Grab latest language file from server using scp

Uncaught SyntaxError: Unexpected identifier

  • Check every object comma / semicolon put correctly

When click button, page no effect (no hang)

  • Make sure all error field display correctly based on backend return

Filter not working

  • Make sure pass searchData instead of inputData
  • dataName = username, dataType = match / like instead of text
  • Go superadmin to make sure searchData pass correctly

When click button page hang, keep show loading icon

  • Make sure language js is up to date
  • Make sure error field display correctly

After apply numberThousand function, showing NaN.00

  • Try parseFloat first for number then only apply numberThousand
  • Make sure backend the data you get is not -, is yes, should do a condition if data == '-', number = 0
  • Check the number that need to parseFloat, if the number applied numberThousand, should use removeComma function then only do calculation.

Uncaught TypeError: Cannot read property 'variable' of undefined

  • Make sure the variable is declare correctly (var variable;)
  • If the variable declared global, make sure when assign variable data in function didn't declare again the variable. Use variable = 1 instead of var variable = 1 in function
  • If it's Cannot read property 'english' of undefined, mean you hav to grab latest language js file from test site using scp.
var declaredVar; $(document).ready(function(){}); function usedFunc() { declaredVar = YOUR_DATA; var declaredVar = YOUR_DATA; }

When click button, run function but immediately refresh page

  • Add a type="button" to the button element.
<button type="button" class="btn btn-primary">Button</button>

Error message didn't clear keep duplicate when call api

  • Add below code to ajaxSend function (general.js)
$(".invalid-feedback").remove(); $(".is-invalid").removeClass("is-invalid");

Same page but backend return two api with same error ID

  • Assume it has two api "command1" and "command2" will pass same error id "tPasswordError"
  1. HTML: One of the tPassword input should change to "tPasswordCmd"
  2. JS: You should pass the api name to showCustomErrorField function (general.js) [ajaxSend Func]
  3. JS: In showCustomErrorField function, you will need to do a condition. If api equal to command2, the error id should replace to tPasswordCmd

HTML

<input id="tPasswordCmd" type="text" class="form-control" />

ajaxSend Function

showCustomErrorField(obj.data.field, obj.statusMsg, val['command']);

showCustomErrorField Function

function showCustomErrorField(data, statusMsg, command) { if(Array.isArray(data)) { $.each(data, function(key, array) { var id = ''; var msg = ""; $.each(array, function(k, v) { if(k == 'id') id = v; else if(k == 'msg') msg = v; if(id == '') return; }); if(id == '') return; else if(msg && msg.length > 0) { switch(command) { case "command2": errorDisplay(id.replace("Error", "Cmd"),msg); break; default: errorDisplay(id.replace("Error", ""),msg); } } }); } else { // API params checking error errorDisplay(data.replace("Error", ""),statusMsg); } }
